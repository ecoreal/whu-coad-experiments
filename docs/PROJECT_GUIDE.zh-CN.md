# 项目指南

[English](PROJECT_GUIDE.md)

本文档用于说明 WHU COAD Experiment 的项目结构与设计思路，帮助读者理解各 Vivado 工程之间的关系，以及单周期 CPU 实验的组成方式。

## 1. 项目定位

WHU COAD Experiment 是一个面向计算机组成与体系结构设计实验的 Verilog HDL 教学项目仓库。项目并不只面向软件仿真，而是围绕 FPGA 板级观察展开：

- 使用开关 (`sw_i`) 选择显示模式、执行速度、复位入口地址和内部数据观察对象。
- 对板载时钟进行分频，得到便于人工观察的 CPU 时钟。
- 使用八位数码管作为主要输出设备。
- 使用 Vivado 分布式存储器 IP 作为指令 ROM，并通过 `.coe` 文件初始化指令内容。

仓库保留重新打开与重新生成工程所需的源码和配置文件，Vivado 生成的构建产物不纳入版本控制。

## 4. 单周期 CPU 架构

CPU 实验使用 `SCPU_TOP` 作为顶层集成模块。整体结构遵循典型单周期数据通路：

1. `PC` 给出当前指令地址。
2. 指令 ROM IP (`dist_mem_gen_10`) 根据 `12.21test.coe` 读取指令。
3. `SCPU_TOP` 从指令中拆分 opcode、funct 字段、寄存器编号和立即数字段。
4. `ctrl` 对指令进行译码并生成控制信号。
5. `EXT` 生成对应类型的立即数。
6. `RF` 读取源寄存器，并在写回阶段更新目标寄存器。
7. `alu` 计算算术结果并生成 Zero 标志。
8. `dm` 处理数据存储器读写。
9. `NPC` 计算下一条指令地址。
10. `seg7x16` 将选定的内部数据输出到八位数码管，便于观察。

该设计刻意保持紧凑，重点服务于课堂实验中的结构理解和硬件可观察性。它强调 CPU 各组成部件如何连接，以及如何在 FPGA 上查看内部信号。

## 5. CPU 模块职责

### `SCPU_TOP`

`SCPU_TOP` 是 CPU 实验的顶层模块，负责时钟分频、连接各数据通路模块、拆分指令字段、选择显示数据，并将开关输入映射到观察行为。

重要行为：

- `sw_i[15]` 选择分频后的 CPU 时钟速度。
- `sw_i[1]` 控制是否允许指令步进。
- `sw_i[14:11]` 选择显示的内部数据来源。
- `sw_i[13]`、`sw_i[12]`、`sw_i[11]` 分别用于切换寄存器、ALU 和数据存储器观察值。
- `sw_i[0]` 在原始段码显示模式和十六进制数据显示模式之间切换。

### `ctrl`

`ctrl` 负责解码指令字段并生成数据通路控制信号：

- `RegWrite` 控制寄存器堆写回。
- `MemWrite` 控制数据存储器写入。
- `EXTOp` 选择立即数扩展格式。
- `ALUOp` 选择 ALU 操作。
- `ALUSrc` 选择 ALU 第二操作数来自寄存器还是立即数。
- `DMType` 选择字节、半字或字形式的存储器访问。
- `WDSel` 选择写回数据来自 ALU、数据存储器或 PC+4。
- `NPCOp` 选择顺序执行、分支、`jal` 或 `jalr` 的下一条 PC 行为。

### `PC` 与 `NPC`

`PC` 保存当前指令地址。复位时，可通过部分开关组合把 PC 设置到预设入口地址，便于观察不同指令演示片段。

`NPC` 负责计算下一条指令地址：

- 普通顺序执行时为 `PC + 4`。
- 分支和 `jal` 时为 `PC + immout`。
- `jalr` 时为 `aluout`。

### `EXT`

`EXT` 将指令中的立即数字段转换为 32 位有符号立即数，支持 CPU 实验使用的 I-type、S-type、B-type 和 J-type 格式。

### `RF`

`RF` 是包含 32 个寄存器的寄存器堆，提供两个读端口和一个写端口。零号寄存器读出恒为零，并通过写地址判断避免写入零号寄存器。

### `alu`

`alu` 执行当前实验所需的算术运算，并生成 Zero 标志。Zero 标志会被控制器用于分支判断。

### `dm`

`dm` 是按字节寻址的数据存储器。它支持字、半字、字节访问，并在读取时处理符号扩展或零扩展。写入时会根据访问类型更新对应字节。

### `seg7x16`

`seg7x16` 用于动态扫描八位数码管，支持两种显示模式：

- 十六进制模式：每 4 bit 数据解码为一个数码管字符。
- 原始段码模式：每 8 bit 数据直接作为一个数码管段码。

## 6. 显示与开关交互

该设计把数码管作为内部信号观察器。在 CPU 项目中，当 `sw_i[0] == 0` 时，显示数据由 `sw_i[14:11]` 选择。

| 开关组合 | 显示数据 |
| --- | --- |
| `sw_i[14:11] = 1000` | 当前指令字 |
| `sw_i[14:11] = 0100` | 寄存器堆观察值 |
| `sw_i[14:11] = 0010` | ALU 观察值 |
| `sw_i[14:11] = 0001` | 数据存储器观察值 |
| 其他组合 | 默认显示值 |

当 `sw_i[0] == 1` 时，`SCPU_TOP` 显示 `LED_DATA` 中预设的原始数码管图案。

## 7. 存储器初始化文件

指令 ROM 使用 COE 文件初始化：

- `Test_8_Instr1.coe` 用于 ROM 演示项目。
- `12.21test.coe` 用于 `12.21test` CPU 项目。

这些文件是可复现实验配置的一部分，因此保留在仓库中。使用 Vivado 打开工程时，Vivado 可以根据 `.xci` 配置和这些 COE 文件重新生成 IP 输出产物。

## 8. 推荐阅读顺序

如果从源码角度学习，建议按以下顺序阅读：

1. `COAD_test/seg7x16.v`
2. `COAD_test/RF.v`
3. `COAD_test/alu.v`
4. `COAD_test/dm.v`
5. `COAD_test/EXT.v`
6. `COAD_test/NPC.v`
7. `COAD_test/PC.v`
8. `COAD_test/ctrl.v`
9. `COAD_test/SCPU_TOP.v`

这个顺序先阅读相对独立的基础模块，最后再阅读顶层集成。

## 9. 在 Vivado 中重新生成

使用 Vivado 2018.3 或兼容版本打开对应 `.xpr` 文件。如果 Vivado 提示 IP 核缺少生成产物，可对该 IP 执行 **Generate Output Products**，然后继续综合、实现和比特流生成。

仓库有意不包含 `.runs`、`.cache`、`.sim`、`.hw`、`.ip_user_files`、报告、检查点和比特流等生成文件。这些文件与本地环境相关，应该在使用者本机重新生成。
